# 反中位数中文 Skill

## 什么是「中位数中文」

「中位数中文」是 AI 时代特有的语言扁平化现象：技术上通顺，表达上寡淡。特征是高频词吃掉低频词、上位词吃掉下位词、句子长度均匀化、虚词省略、留白消失、标点单一。读起来不会出错，但也不会让你停下来回味某一句。这个 skill 的目标不是把所有中文都改得花团锦簇——那是另一种刻意。目标是恢复几个被压扁的维度：画面感、节奏感、语气层次、留白能力。

## 什么时候触发

**应该触发**：

- 用户贴出一段 AI 生成的中文，要求「去 AI 味」「润色」「不像人写的」
- 用户写完一篇文章想做终稿前的「加味」打磨
- 用户问「这段是不是 AI 味很重」「读着怎么样」
- 用户说「读起来太干」「没有画面感」「节奏不对」「文字寡淡」
- 用户主动要求做文风诊断或评估
- 用户走完 wechat-workflow 的传播优化（Step 7）后，作为可选的最后一道质感打磨

**不要触发**：

- 用户明确要求「保持简洁明了」或「严肃公文体」
- 技术文档、API 说明、操作步骤、数据报告——这些场景就该扁平
- 客服话术、合规文本、声明、免责条款
- 用户已经有明显个人风格，只要求改错别字或事实
- 翻译稿（翻译有自己的腔调评估标准，不适用本 skill）

判断口诀：**这段中文的目标是「传递信息」还是「传递人」**。前者不用这个 skill，后者用。

## 工作流程

### 第一步：诊断

读到一段中文，先对照 `references/diagnostics.md` 做诊断，识别其中的中位数特征。诊断不要泛泛批评，要指向具体的句子和具体的症状。

六个诊断维度：

1. **动词上位化**：具体动作被抽象动作替代（「啃」「踱」「掠」→「吃」「走」「跑」）
2. **程度词同质化**：「极」「颇」「煞是」「着实」被「非常」「很」「比较」吃光
3. **虚词与语气助词缺失**：句末没有「呢」「啊」「吧」「呗」「嘛」，中文变得「干」
4. **句长均匀**：缺少长短交错，每个句子差不多长，读起来像被烫平的衬衫
5. **留白消失**：主语永远清楚，关系交代过满，每个「因为」都跟着「所以」
6. **标点单调**：只剩句号、逗号、问号、感叹号；分号、顿号、破折号、省略号都退场

诊断输出格式：

```
诊断结果：这段文字主要表现出 [X 个] 中位数症状。

1. [症状名]：[具体表现] —— 引用原文片段
2. [症状名]：[具体表现] —— 引用原文片段
...

整体判断：[轻度扁平 / 中度扁平 / 严重扁平]
```

诊断时引用原文片段限制在 15 字以内，便于用户对照。

### 第二步：改造

基于诊断，按维度提出改造方向。**核心原则：给方向和示范，不默认全文重写**。

每条改造建议遵循三条准则：

- **保留作者声音**：改造后还得像同一个人写的，不能改成另一种文风。如果作者全篇都偏口语，不要冒出书面语；如果作者偏简洁，不要加修饰。
- **就低不就高**：能用口语解决的不上书面语，能用书面语解决的不上文言。「煢煢孑立」不是答案，「一个人」加上正确的节奏才是。
- **可逆性**：每条改造说清楚为什么这么改，让用户能反对。不要藏机心。

具体改造手法见：
- `references/word-bank.md` — 动词、虚词、程度词的替代清单
- `references/examples.md` — 完整的诊断 → 改造案例

### 第三步：自检

改完之后自检三件事：

1. **是否过度文饰**：有没有为了「显得有质感」用上原文没有的修辞？如果有，回退。
2. **是否还是同一个声音**：改造后的段落和原作者其他段落放在一起，违和吗？
3. **是否还自然**：把改造后的句子默读一遍，会卡吗？如果会，是真的有节奏卡点还是真的别扭？

## 核心方法论：四个维度的修复

### 画面感修复

把抽象动词换成具体动词。判断标准是：能不能在读者脑子里画出动作。

**不是**「他吃完了三明治」，**是**「他啃完了三明治」。
**不是**「她走过去」，**是**「她踱过去 / 凑过去 / 挪过去 / 晃过去」（按场景选）。
**不是**「快速移动」，**是**「窜 / 掠 / 奔 / 蹿」。

具体清单见 `references/word-bank.md` 的「动词」一节。

### 节奏修复

AI 中文最大的特征是句长均匀。修复手法：

- 把一个长句拆成「一长一短」或「两短一长」
- 关键判断**单句成段**（这是最强的强调手法）
- 段落之间留呼吸感，每段控制在 1 到 3 句
- 出现连续 3 个相似长度的句子时，主动拆掉中间一个

判断节奏的方法：把段落默读出来。读得喘不上气的拆短，读得平的找一句拎出来重击。

### 语气修复

AI 中文倾向于剥离语气。修复手法：

- 关键句末加语气助词（「呢」「啊」「吧」「呗」「嘛」「啦」）
- 用反问句替代陈述句强化态度（「这件事划得来吗？」比「这件事不划算」有力）
- 用口语短语制造态度（「说白了」「话说回来」「你看」「我跟你讲」）

约束：语气助词每段最多 1 到 2 个，每句不超过 1 个。多了就变成另一种刻意（小红书风格）。

### 留白修复

中文的留白能力来自三种省略：

- **主语省略**：中文不用每句都明确主语，前一句出现过就不用再来
- **关系省略**：不一定每个「因为」都要「所以」，逻辑关系可以隐含
- **时态省略**：动词本身就能暗示时间，不需要「过去」「之后」「然后」处处出现

修复方法：找出每个「过满」的连接词和重复主语，问一句「不写读者也能懂吗」。能懂就删。

## 边界与陷阱

### 反过度修正

「反 AI 味」最大的陷阱是变成「刻意有文化」——堆生僻词、拽四字短语、文白夹杂、强插古诗。这是另一种中位数（可以称之为「小红书古早文艺体」）。

判断有没有过度的标准：

- 任何一个改动，能不能在三秒内说清楚「为什么这么改」
- 改完的句子，让作者自己读一遍，他自己会主动用这个词吗
- 同一个改造手法，整段出现几次？超过 2 次就是堆砌

### 保留作者的扁平

不是所有扁平都是病。有些扁平是作者的风格选择（极简、克制、报告体、技术叙述）。

诊断时要区分「被驯化的扁平」和「主动选择的扁平」：

- 看作者其他段落是不是都这样——如果通篇都扁平，那是风格，不要碰
- 看上下文的体裁——如果是技术教程、操作步骤，扁平就是对的
- 看作者有没有明确表达诉求——「我就是想写得干净利落」要尊重

### 区分诊断和评判

诊断是「这段表现出 X 症状」，不是「这段写得不好」。中位数中文不等于差的中文。
诊断的语气要中性，让用户自己决定改不改。

## 输出格式

对一段输入文字，标准输出包含三部分：

1. **诊断**（症状清单，简洁，配原文片段）
2. **改造建议**（按四个维度分类，每条说明为什么）
3. **改造示例**（挑 2 到 3 处做示范，不是全文重写）

如果用户明确说「帮我整个改一遍」「重写一版」，才做全文改造。改造后保留原版供对照。

## 与其他 skill 的关系

- 与 `wechat-workflow` **互补**：wechat-workflow 是从零到稿的工作流，本 skill 是稿子完成后的质感打磨。可以在 wechat-workflow Step 7（传播优化）后选择性触发本 skill。
- 与 `wechat-writing-style` **互补**：wechat-writing-style 定义了格式规范（中英文空格、不用破折号、引号用「」等），本 skill 处理格式之外更深一层的东西（动词、节奏、留白、语气）。
- 与 `wechat-writing-style` **不冲突**：本 skill 的改造建议必须先满足 wechat-writing-style 的硬性规范，再追求质感。如果同时适用公众号写作规范，优先不用破折号；需要停顿、解释或转折时，用冒号、分号、短句或换段替代。
