# 改造前后对照案例

本文件供 SKILL.md 第二步参考，展示完整的诊断 → 改造流程。每个案例包括：原文、诊断、改造建议、改造示例。

---

## 案例 1：AI 生成的产品介绍

### 原文（中位数中文，严重扁平）

> 在当今数字化转型的浪潮中，越来越多的企业开始意识到提升用户体验的重要性。我们的产品提供了一站式的解决方案，能够帮助企业快速实现数字化转型，提升运营效率，降低运营成本。通过先进的技术架构和丰富的功能模块，我们的产品已经服务了众多行业的客户，获得了广泛的好评。

### 诊断

```
这段文字主要表现出 5 个中位数症状。

1. 动词上位化：「实现」「提升」「降低」「提供」全是抽象动作，无画面感
2. 程度词同质化：「越来越多」「广泛」「丰富」「众多」，程度全是泛泛的
3. 留白消失：每句都有完整主谓宾，且主语反复出现（「我们的产品」「企业」）
4. 句长均匀：四个句子都在 25 到 40 字之间，节奏一致
5. 标点单调：全文只有句号和逗号

整体判断：严重扁平。这是典型的 AI 营销话术输出。
```

### 改造建议

按四个维度：

- **画面感**：换掉「实现」「提升」「降低」这类空动词，要么删掉，要么换成具体动作
- **节奏**：拆开长句，让一两个短句拎出来
- **语气**：加 1 到 2 处口语，破除营销腔
- **留白**：删掉重复的「我们的产品」「企业」，让逻辑隐含

### 改造示例

> 数字化转型这个词，企业听了不下一万遍。落地难在哪？体验、效率、成本，全要一把抓。
>
> 我们这套方案，就是冲着这三件事来的。
>
> 已经跑通了若干行业，客户反馈不错。

**改了什么**：

- 「在当今...的浪潮中」整句删掉，无信息量
- 「越来越多企业意识到...的重要性」换成具体场景「企业听了不下一万遍」
- 「能够帮助...提升...降低...」换成口语「一把抓」
- 「先进的技术架构和丰富的功能模块」整句删掉，是套话
- 「广泛的好评」换成「反馈不错」，去掉夸张
- 第三段单句成段，加强收尾

**字数**：原文 134 字 → 改造 78 字。信息密度反而更高。

## 案例 2：AI 写的旅行散文

### 原文（中度扁平）

> 走在京都的街道上，我感受到了一种宁静的氛围。两旁的房屋古色古香，让人仿佛回到了过去的时光。我看到很多游客在拍照，他们的脸上洋溢着幸福的笑容。这里的一切都很美好，让我久久不愿离去。

### 诊断

```
这段文字主要表现出 4 个中位数症状。

1. 动词上位化：「走」「看」「感受」「洋溢」，可以更具体
2. 程度词同质化：「很」「非常」省略了但用了「久久」「很多」，仍偏抽象
3. 留白消失：「我」反复出现，「让人」「让我」交代过满
4. 句长均匀：四句话都在 18 到 22 字

整体判断：中度扁平。是典型的 AI 抒情文体。
```

### 改造建议

- 「走」改成具体的步态动词
- 「看到游客拍照」可以更具体（拍什么？怎么拍？）
- 删掉「我」的重复
- 把「久久不愿离去」这种套话换成具体场景

### 改造示例

> 京都的街上，安静得能听见自己脚步。两旁的老房子贴着墙根站，木格窗都泛着旧色。
>
> 游客举着相机蹲在屋檐下，找一个能把整条巷子框进去的角度，反复按快门。
>
> 走过去半小时，又折回来。

**改了什么**：

- 「我感受到了一种宁静的氛围」→「安静得能听见自己脚步」，把感受改成画面
- 「古色古香」→「贴着墙根站」「木格窗都泛着旧色」，给抽象形容词找具体形象
- 「他们的脸上洋溢着幸福的笑容」→「找一个能把整条巷子框进去的角度，反复按快门」，把抽象的「幸福」换成具体的拍照动作
- 「久久不愿离去」套话整句删，换成「走过去半小时，又折回来」，用行动暗示

**保留了什么**：作者原本的抒情视角和散文节奏没有变，只是把抽象描写换成了具体画面。

## 案例 3：AI 写的工具评测开头（轻度扁平）

### 原文

> 最近我用了一个新的 AI 工具，叫 Claude。这个工具的功能很强大，可以帮助我完成很多任务。我用了大概一周时间，整体感觉还不错，推荐给大家。

### 诊断

```
这段文字主要表现出 3 个较轻的中位数症状。

1. 动词上位化：「用」「完成」「帮助」过于泛泛
2. 程度词同质化：「很强大」「很多」「还不错」全是模糊程度
3. 句长均匀：三句话都在 20 字左右

整体判断：轻度扁平。虽然触发 3 项，但症状都停留在具体性不足，没有明显破坏作者声音。
```

### 改造建议

这种轻度扁平改造起来反而要克制。核心问题是「具体性不够」，不是「文字不漂亮」。改造方向：

- 加一两个具体场景（用它干了什么）
- 把「很不错」换成有判断的话
- 不要为了「有质感」加修辞

### 改造示例

> 最近我用 Claude 用得比较多。一周下来主要干两件事：写公众号长文，跑数据分析脚本。
>
> 长文这块比 ChatGPT 顺手，脚本这块还差点意思。
>
> 整体值得推荐，但别神化。

**改了什么**：

- 「功能很强大」→ 具体说「干两件事：写公众号长文，跑数据分析脚本」
- 「整体感觉还不错」→ 拆成「长文这块...脚本这块...」给出对比判断
- 「推荐给大家」→「值得推荐，但别神化」，加一个反向判断

**字数**：原文 60 字 → 改造 75 字。略长，但每个字都有用。

**保留了什么**：作者的口语风格和「我」的视角没动，只是把空话换成了具体经验。

## 案例 4：什么时候不要改

### 原文（技术文档）

> 这个 API 接受三个参数：query（字符串，必填）、limit（整数，默认 10）、offset（整数，默认 0）。返回 JSON 格式的结果，包含 results 数组和 total 字段。

### 诊断

```
这段文字符合多个「中位数」特征，但不应触发改造。

理由：这是 API 文档，目的是传递信息不是传递人。
扁平、规整、可预测，恰恰是这种文体的优点。
```

### 不改的原因

- 技术文档的目标是清晰，不是有味道
- 加画面感反而会让读者怀疑文档的准确性
- 节奏感对查文档的人没有价值

**判断口诀重申**：这段中文的目标是「传递信息」还是「传递人」？前者不用改。

## 案例 5：作者风格本身偏扁平

### 原文（用户自己写的，但风格本来就极简）

> 今天加班到十点。地铁停了。打车回家。睡了。

### 诊断

```
这段文字虽然全是短句、动词扁平、留白极满，
但不应判定为中位数中文。

理由：作者明显在主动选择极简风格，
不是被算法驯化的扁平。强行加画面或加节奏，
反而会破坏作者的声音。
```

### 不改的原因

- 看上下文：如果作者其他段落也这样，是风格
- 看效果：极简本身就有节奏（短句堆叠的鼓点感）
- 看意图：作者明显不想多说，硬要他多说是越俎代庖

**判断方法**：诊断前先看作者的其他段落或其他作品。如果通篇一致，那是风格，不要碰。

## 改造的总体节奏

回顾这五个案例，可以总结出改造的几个层次：

1. **严重扁平**（案例 1）：大刀阔斧，重写句式和动词
2. **中度扁平**（案例 2）：保留结构，换关键词和画面
3. **轻度扁平**（案例 3）：补具体性和判断，不动文体
4. **不该改**（案例 4、案例 5）：识别出风格选择或文体限制，按兵不动

改造的功夫，一半在动笔，一半在识别什么不该动。
