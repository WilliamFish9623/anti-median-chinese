# 中位数中文诊断清单

本文件供 SKILL.md 第一步「诊断」加载使用。逐条对照原文，识别症状。

---

## 症状 1：动词上位化

**特征**：本来应该用具体动词，却用了上位的抽象动词。

**典型征兆词**：

| 上位词 | 应该考虑的下位词 |
|---|---|
| 走 | 踱、踉跄、徜徉、晃、踱步、挪、蹿、凑 |
| 跑 | 窜、掠、奔、驰、蹿、奔逃 |
| 吃 | 啃、咂、嚼、抿、嗦、撕咬、扒拉 |
| 喝 | 抿、嘬、啜、灌、咕咚 |
| 拿 | 攥、捏、捻、掂、揪、抄 |
| 看 | 瞧、瞅、望、瞥、扫、盯、端详 |
| 说 | 嘟囔、嘀咕、嚷、念叨、絮叨、咕哝 |
| 笑 | 咧嘴、抿嘴、扑哧、噗嗤、嘿笑 |
| 移动 | 挪、蹭、晃、溜、窜、蹿 |
| 接触 | 碰、蹭、贴、抚、按 |

**诊断判断**：原文里有没有「走 / 跑 / 吃 / 喝 / 看 / 说 / 拿」这类大词，且场景明明可以更具体？有就标记症状 1。

**反例（不算症状）**：上下文确实只需要一个泛指动作（如「我每天走两万步」），不必改。

## 症状 2：程度词同质化

**特征**：所有程度都被「非常」「很」「比较」「特别」吃掉。

**单一程度词的可替代选项**：

| 模板 | 替代 |
|---|---|
| 非常 / 很 + 形容词 | 极 / 颇 / 着实 / 煞是 / 委实 / 大 / 真 + 形容词 |
| 非常重要 | 关键 / 要紧 / 紧要 / 至关重要 |
| 非常好 | 极好 / 妙 / 漂亮 / 顶好 |
| 非常困难 | 棘手 / 麻烦 / 不好办 / 难办 |
| 非常喜欢 | 偏爱 / 中意 / 钟爱 / 死磕 |
| 比较 + 形容词 | 略 / 稍 / 还算 / 算得上 |

**诊断判断**：通篇「非常」「很」「比较」出现 3 次以上，且对象不同？标记症状 2。

**警告**：程度词不要全部换成偏书面的，要按上下文选。口语场景下「老好」「贼好」「真好」比「极好」自然。

---

## 症状 3：虚词与语气助词缺失

**特征**：在明显需要口语、人称或态度的文本里，句末没有「呢」「啊」「吧」「呗」「嘛」等语气层次；连接处也只有最常见的转折和递进。

**诊断信号**：

- 叙述视角本来贴近作者或读者，却通篇没有任何句末语气助词
- 「但是 / 不过 / 然而」之外没有别的转折（缺「却」「倒是」「反倒」「偏偏」）
- 「也」「还」「再」「又」之外没有别的副词层次

**反例（不算症状）**：严肃评论、书面散文、技术说明、公文和报告不一定需要句末助词。不要因为没有「呢啊吧嘛」就判定文字干。

**修复方向**：

| 干巴版 | 加语气版 |
|---|---|
| 这件事不划算 | 这件事划得来吗？/ 这件事啊，不划算 / 这件事不划算呗 |
| 我不同意 | 我可不同意 / 我倒不同意 / 我不同意啊 |
| 他没来 | 他偏没来 / 他倒是没来 |
| 这个方案行不通 | 这个方案行不通呢 / 这方案哪行得通 |

**约束**：每段最多 1 到 2 个语气助词，分散使用，不要堆在一起。

---

## 症状 4：句长均匀

**特征**：相邻句子长度差不多，读起来像被烫平的衬衫。

**诊断方法**：

1. 数一段里每句的字数
2. 如果连续 3 句字数差距小于 5 个字，就是症状 4
3. 如果整段所有句子都在 15 到 25 字之间，是典型症状 4

**修复方向**：

- 把一个长句拆成「短句 + 长句」或「长句 + 短句」
- 找一句关键判断**单句成段**
- 段落首尾用短句锚定，中间允许长句展开

**示例对照**：

**症状 4 的版本**：
> 这件事我考虑了很久，最后还是决定试一试。失败了大不了重来，反正不会有更糟的结果。年轻的时候不试，老了一定会后悔。

（三句话都 15 字左右，节奏一样）

**修复版**：
> 这件事我考虑了很久。最后还是决定试一试——失败了大不了重来，反正不会有更糟的结果。
>
> 年轻不试，老了后悔。

（短句开头 + 长句展开 + 短句收尾，节奏出来了）

## 症状 5：留白消失

**特征**：每句主语清楚，每个逻辑关系挑明，每个时间状语完整。读者不需要的信息全交代了。

**诊断信号**：

- 同一段里「我」「我们」「他」反复出现，每句都来一遍
- 「因为...所以...」「虽然...但是...」成对出现
- 「然后」「接着」「之后」频繁出现
- 「这个问题」「这种情况」「那件事」反复指代

**修复方向**：

| 过满版 | 留白版 |
|---|---|
| 因为下雨了，所以我没去 | 下雨了，没去 |
| 虽然他很努力，但是没成功 | 努力归努力，没成 |
| 我做完了作业，然后我去吃饭，吃完饭我去睡觉 | 做完作业，吃饭，睡觉 |
| 我觉得这个方案有问题 | 这方案有问题 |

**核心方法**：每个连接词、每个重复主语，问一句「不写读者也能懂吗？」能懂就删。

## 症状 6：标点单调

**特征**：通篇只有句号、逗号、问号、感叹号。

**诊断信号**：

- 全文没有分号
- 全文没有顿号（并列只用逗号）
- 全文没有破折号
- 全文没有省略号
- 引号统一是 ""，没有「」（这条按 wechat-writing-style 规范处理）

**修复方向（按场景）**：

| 标点 | 使用场景 |
|---|---|
| 分号 ； | 并列分句之间，比逗号重、比句号轻 |
| 顿号 、 | 同级并列的词，比逗号更紧凑 |
| 破折号 —— | 解释、转折、思路跳跃；占两个字宽。若公众号规范禁用破折号，改用冒号、分号、短句或换段 |
| 省略号 …… | 欲言又止、留白、列举未尽 |
| 「」 | 直接引语、需要强调的概念 |

**警告**：

- 中文破折号是 ——（两个字宽的连贯横线），不是英文 em dash（—）
- 省略号是 ……（六个点占两格），不是英文三个点（...）
- 如果同时适用 wechat-writing-style 或公众号硬规范，优先不用破折号
- 不要为了「显得有标点」硬塞，每个标点要有用

## 严重程度判断

诊断完成后，给整段一个整体判断：

| 程度 | 标准 |
|---|---|
| 轻度扁平 | 触发 1 到 2 个症状；或触发 3 个但都很浅，主要是具体性不足 |
| 中度扁平 | 触发 3 到 4 个症状，且已经影响画面、节奏或判断 |
| 严重扁平 | 触发 5 到 6 个症状，是典型 AI 输出味 |

判断的目的不是机械评分，是让用户对修改投入有预期。轻度扁平可能调一两个动词就够了，严重扁平需要整段重写。
